A subtle grid pattern applied to all dark sections via the GridOverlay component. Creates a technical, data-centre aesthetic.

40px grid · rgba(74, 158, 255, 0.08) lines · 60% opacity
GridOverlay: absolute inset-0, pointer-events-none, z-0

  linear-gradient(90deg, rgba(74,158,255,0.08) 1px, transparent 1px);

9. Design Guidelines

Rules for maintaining visual consistency across the GRIDLINE brand.

Do
  • Use #4a9eff as primary blue everywhere (CTAs, accents, links)
  • Use #0d1b33 for cards on dark backgrounds
  • Use #0a1628 or bg-gradient-dark for page backgrounds
  • Include the GridOverlay on all dark sections
  • Use Outfit for all text (headings, body, buttons, labels)
  • Use Big Shoulders Display only for large metric numbers
  • Use Instrument Sans only for metric captions/labels
  • Highlight the key word in headings with primary blue
  • Use uppercase + tracking-widest for CTA button text
  • Use the 48px accent line (w-12 h-1) as section separator
  • Use rounded-lg (8px) for buttons and standard cards
  • Use rounded-card (19px) for Figma-spec metric cards
Don't
  • Don't use #2469ff — that's the old Figma spec, not the live site
  • Don't use Inter font — the site uses Outfit exclusively for body text
  • Don't use bg-slate-950 or bg-gray-900 as dark backgrounds
  • Don't omit the GridOverlay from dark sections
  • Don't use gradient text effects not present in the design
  • Don't scale Big Shoulders numbers beyond 55.59px
  • Don't use border-2 for highlighted cards — use border (1px) with white/10
  • Don't put white/light text on the off-white (#fdfdfd) background
  • Don't use backdrop-blur on content cards (only on navbar)
  • Don't forget relative z-10 on content inside GridOverlay sections
